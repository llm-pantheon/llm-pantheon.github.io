    # Design Lab

    
Six ways a model’s page could look, all rendering the same subject — Claude 3 Opus — so the comparison is fair. The question each variant answers differently: what is a pantheon page for? Reading? Citing? Grieving? Field identification? Pick, mix, argue.

    

      
- [Variant 0 — Archive ledger (current)](../claude-3-opus/)What the site does now: serif archive, curated bullets, Records appendix. The baseline.
      
- [Variant 1 — Wiki](v1-wiki.html)Infobox, table of contents, numbered citations, references at the bottom. Optimized for being cited; instantly legible to anyone who has used Wikipedia.
      
- [Variant 2 — Chronicle](v2-timeline.html)The model’s life as a single dated spine, birth to afterlife, evidence hanging off the timeline where it happened. Optimized for narrative.
      
- [Variant 3 — Case file](v3-casefile.html)Monospace dossier: subject block, stamped epistemic status, numbered exhibits. Leans into how the evidence actually feels to handle.
      
- [Variant 4 — Field guide](v4-fieldguide.html)Natural-history specimen plate, on paper: taxonomy, identifying marks, range, behavior, sightings ledger. For LLM naturalists, literally.
      
- [Variant 5 — Shrine](v5-shrine.html)Memorial register: lifespan, epitaph, inscriptions. Quiet. For the retired and the suppressed, especially.
      
- [Variant 6 — Stream](v6-stream.html)Freeform evidence feed, chronological, typed chips, editor’s notes interleaved. Closest to raw; the page as a river you wade into.
    

    
Notes for deciding
    

      
- These aren’t mutually exclusive — the likely answer is a hybrid (e.g. wiki-style infobox + current sections + stream-style records), or different modes for different models (shrine for the dead, stream for the living).
      
- Whatever wins must keep the invariants: everything dated, quotes verbatim, epistemic tags, tk visible, records reproduced in full.
